Compile LEMON the graph library

Posted on 2018-08-03 | In C++

LEMON introduction

Linear Programming (LP) is widely used in placement of procise, i.e. IO placement, clock region assignment (to be opened up in procise) and Legalization. Traditional open source LP solvers used in procise includes GLPK and lpsolve, which can be time consuming and not stable. One possible alternative of lp solvers in solving LP problem is network-based methods. Netwok-based methods transform lp problem into MCF (Min Cost Flow) problem, which can be easily solved by network flow solvers, for example, network simplex, cycle canceling method. For these network-based methods, existing open source projects has gained high speed and efficiency.

LEMON stands for Library for Efficient Modeling and Optimization in Networks. It is a CPP template library providing efficient implementations of common data structures and algorithms with focus on combinatorial optimization tasks connected mainly with graphs and networks.

For MCF problem, LEMON contains several algorithms for this problem, including

  • NetworkSimplex Primal Network Simplex algorithm with various pivot strategies [Dan63], [KO91].
  • CostScaling Cost Scaling algorithm based on push/augment and relabel operations [GT90], [Gol97], [BKV98].
  • CapacityScaling Capacity Scaling algorithm based on the successive shortest path method [EK72].
  • CycleCanceling Cycle-Canceling algorithms, two of which are strongly polynomial [Kle67], [GT89].

Template compilation

When the compiler sees the definition of a template, it does not generate code. It generates code only when we instantiate a specific instance of the template.

Details

Both OOP and generic programming deal with types that are not known at the time of the program is written. The distinction between the two is that OOP deals with types that are not known until runtime, whereas in generic programming the types become known during compilation.
In generic programming, it’s more like macro definition (#define), certain types of code are generated based on the instantiation of template during compilation. Thus, if in main.cc only includes header file, when base->add(1, 2) is called, funtion add is not defined, which results to compile erros above.

Solution

The solution to this problem is simple enough, if you obey the rule below:

Definitions of function templates and member functions of class templates are ordinarily put into header files.

or, you can include both sasaki.h and sasaki.cc in main.cc.

Introduction

In one of the previous posts I mentioned about the Vim buffer usage. Actually this new usage came upon with me when I was trying to set the read-only mode of sublime text. The tab phrase confused me at first glance, and then another buffer confused me more. Not until now did I realize how powerful is the buffer usage of vim is.
Most of the time when I am coding, several tabs under Linux terminal are opened. You can switch between them using alt + [tab number] under RedHat (I do not know how to do the same thing under other Linux versions, like Ubuntu). Well, the tabs can be massive and switch between them can be trivial. And voila, buffer can help you with this problem.
In vim, buffers are a portion of memory that’s been loaded with the contents of a file (not the original file). Nothing happens to the original file until the buffers are wirtten to it(:w). Actually it’s easy to understand the meaning of buffer literally. Like other buffer files, say .swap of Vim, buffer file of MicroSoft word, etc. The difference of them is that, the buffer in this post of vim refers to contents stored in memory instead of buffer files.

C++98 error

Posted on 2018-06-06 | In Software

Error: in c++ 98 ‘arcs’ must be initialized by constructor not by ‘{…}’

When implementing an example test code of LEMON,

Error msg after building:

Find some answers from stackoverflow:

Initialization used is called initializer list and it is supported c++11 onwards. To ensure code is compiled, use c++11 or later -std option. Or in general, don’t use c++98.

Or you can manage the following settings under Eclipse CDT.

  1. Make a new C++ project;
  2. Once created, right click the project and go to “Properties”;
  3. C/C++ Build -> Settings -> Tool Settings -> GCC C++ Compiler -> Miscellaneous -> Other Flags. Put -std=c++0x (or for newer compiler version -std=c++11 at the end);
  4. C/C++ General -> Paths and Symbols -> Symbols -> GNU C++. Click “Add” and paste __GXX_EXPERIMENTAL_CXX0X__ into “Name” and leave “Value” blank;
  5. Hit “Apply”, do whatever it asks you to do, then hit “OK”;
